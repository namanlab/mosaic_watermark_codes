# MOSAIC — a *set-based* LLM watermark (content-addressed fountain codes + belief propagation)

Every prior multi-bit text watermark reads the
token stream as an **ordered sequence** (a time signal). MOSAIC treats it as an
**unordered bag of parity samples**. That single change of viewpoint is what
makes it robust to rearrangement, insertion, and deletion *by construction*.

## The idea in one picture

```
   message m (k bits)
        │
        │  each generated token, keyed by an ORDER-CANONICAL hash of its
        │  local context (the *sorted* previous-token window), commits to:
        ▼
   a random PARITY CHECK over the message bits:  m[i] ⊕ m[j] ⊕ m[l]
        │   (this is a fountain / LT code generated by the text itself)
        ▼
   bias the token toward green if that parity (⊕ chip) = 1, else red; sample
        ▼
   TEXT  ──►  detector recomputes every token's check from its context,
             reads its colour → a soft observation of that parity,
             collects ALL of them (order irrelevant) and runs
             BELIEF PROPAGATION over the Tanner graph → recovers m
```

Two design choices do the heavy lifting:

1. **Content-addressed, order-canonical keying.** A token's check / chip / green
   mask depends only on the *sorted multiset* of its preceding window — never on
   its position. Rearranging blocks of text leaves every interior token's vote
   untouched; only the few tokens straddling a new seam are disturbed.
2. **Belief propagation, not Viterbi.** Recovering a message from an *unordered*
   bag of noisy parity samples requires a decoder that ignores arrival order.
   Viterbi is a trellis — inherently sequential. Belief propagation over a
   Tanner graph is order-agnostic, and being a **fountain (rateless)** code,
   deletions simply remove equations instead of breaking a fixed codeword.

## Usage

```python
from src.mosaic import MosaicWatermark

wm = MosaicWatermark(model_name="gpt2", secret_key="key", k_bits=32, delta=6.0)
msg = [1,0,1,1,0,0,1,0]*4
res = wm.generate("The future of renewable energy", msg, max_new_tokens=200)

det = wm.detect_text(res["text"], prompt="The future of renewable energy")
print(det["message"] == msg, det["p_value"])

# detection needs only the tokenizer + key, no model:
wm2 = MosaicWatermark(model_name="gpt2", load_model=False, k_bits=32)
print(wm2.detect_text(suspect_text)["message"])

# quality comparison: WITH vs WITHOUT bit-marking, scored by perplexity
plain  = wm.generate("The future of renewable energy", msg, watermark=False)
marked = wm.generate("The future of renewable energy", msg, watermark=True)
print("ppl plain   :", wm.perplexity("The future of renewable energy", plain["generated_ids"]))
print("ppl marked  :", wm.perplexity("The future of renewable energy", marked["generated_ids"]))
```
